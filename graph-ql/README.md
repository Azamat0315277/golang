## What This Project Does

This is a backend API that lets you **create, read, update, and delete job listings** via GraphQL. It exposes a single endpoint (`/query`) and includes a built-in browser playground for testing queries without any extra tooling.

---

## Tech Stack

| Tool | Purpose |
|---|---|
| [Go](https://go.dev/) | Backend language |
| [gqlgen](https://gqlgen.com/) | GraphQL server code generator for Go |
| [MongoDB](https://www.mongodb.com/) | Database |
| [mongo-driver](https://www.mongodb.com/docs/drivers/go/current/) | Official Go MongoDB driver |

## Project Structure

```
graph-ql/
├── server.go                    # Entry point — starts the HTTP server
├── go.mod                       # Go module and dependency definitions
├── gqlgen.yml                   # gqlgen code generation config
│
├── graph/
│   ├── schema.graphqls          # GraphQL schema (source of truth — edit this)
│   ├── schema.resolvers.go      # Your resolver logic lives here
│   ├── resolver.go              # Resolver struct definition
│   └── generated.go             # Auto-generated by gqlgen (do not edit)
│
├── graph/model/
│   └── models_gen.go            # Auto-generated Go structs from the schema (do not edit)
│
├── database/
│   └── database.go              # MongoDB connection and all DB operations
│
├── mock_data.js                 # MongoDB seed script for local testing
└── graphql_examples.graphql     # Ready-to-run query and mutation examples
```

> **Rule of thumb:** You will mostly work in `graph/schema.graphqls`, `graph/schema.resolvers.go`, and `database/database.go`. Never manually edit `generated.go` or `models_gen.go`.

## Prerequisites

Make sure you have these installed before starting:

- **Go 1.21+** → [download](https://go.dev/dl/)
- **MongoDB** running locally on port `27017`
  - Default credentials: username `admin`, password `password123`
  - Quickest way: run it via Docker (see below)

## Quick Start

### 1. Clone the repository

```bash
git clone <your-repo-url>
cd graph-ql
```

### 2. Start MongoDB (Docker — easiest option)

```bash
docker run -d \
  --name mongodb \
  -p 27017:27017 \
  -e MONGO_INITDB_ROOT_USERNAME=admin \
  -e MONGO_INITDB_ROOT_PASSWORD=password123 \
  mongo:7
```

If you already have MongoDB installed locally, make sure it is running and that you have a user with the credentials above, or update the connection string in `database/database.go`:

```go
// database/database.go, line 16
var connectionString string = "mongodb://admin:password123@localhost:27017/?authSource=admin"
```

### 3. Install dependencies

```bash
go mod tidy
```

### 4. Run the server

```bash
go run server.go
```

You should see:

```
connect to http://localhost:8080/ for GraphQL playground
```

### 5. Open the playground

Go to **http://localhost:8080/** in your browser. You'll see an interactive GraphQL playground where you can type and run queries.

## Using the API

The server runs on **http://localhost:8080**

| URL | Purpose |
|---|---|
| `http://localhost:8080/` | Interactive GraphQL Playground |
| `http://localhost:8080/query` | GraphQL API endpoint |

See `graphql_examples.graphql` for a full set of copy-paste-ready examples. Here are the most common ones:

### Get all job listings

```graphql
query {
  jobs {
    _id
    title
    company
    description
    url
  }
}
```

### Get a single job by ID

```graphql
query {
  job(id: "PASTE_AN_ID_HERE") {
    _id
    title
    company
    description
    url
  }
}
```

### Create a job listing

```graphql
mutation {
  createJobListing(input: {
    title: "Go Developer"
    description: "Build APIs with Go and PostgreSQL."
    company: "GopherCo"
    url: "https://gopherco.io/careers"
  }) {
    _id
    title
  }
}
```

### Update a job listing (partial — only include fields you want to change)

```graphql
mutation {
  updateJobListing(id: "PASTE_AN_ID_HERE", input: {
    title: "Senior Go Developer"
  }) {
    _id
    title
    company
  }
}
```

### Delete a job listing

```graphql
mutation {
  deleteJobListing(id: "PASTE_AN_ID_HERE") {
    deletedJobId
  }
}
```

## GraphQL Schema

The full schema is in `graph/schema.graphqls`:

```graphql
type JobListing {
  _id: ID!
  title: String!
  description: String!
  company: String!
  url: String!
}

type Query {
  jobs: [JobListing!]!       # list all jobs
  job(id: ID!): JobListing!  # get one job by ID
}

type Mutation {
  createJobListing(input: CreateJobListingInput!): JobListing!
  updateJobListing(id: ID!, input: UpdateJobListingInput!): JobListing!
  deleteJobListing(id: ID!): DeleteJobResponse!
}
```

`UpdateJobListingInput` fields are all optional — you only need to include the fields you want to change.

## How the Code Is Organized

Understanding the data flow helps when adding features:

```
HTTP Request
    │
    ▼
server.go          ← sets up the HTTP server and routes
    │
    ▼
graph/generated.go ← gqlgen-generated routing (do not touch)
    │
    ▼
graph/schema.resolvers.go  ← YOUR logic: connects GraphQL operations to DB calls
    │
    ▼
database/database.go       ← MongoDB queries (CRUD operations)
    │
    ▼
MongoDB (graphql-job-board database, jobs collection)
```

When you want to **add a new field or operation:**
1. Edit `graph/schema.graphqls`
2. Run `go run github.com/99designs/gqlgen generate` to regenerate code
3. Implement the new resolver in `graph/schema.resolvers.go`
4. Add the DB logic in `database/database.go`

## Regenerating GraphQL Code

Whenever you change the schema (`schema.graphqls`), regenerate the Go code:

```bash
go run github.com/99designs/gqlgen generate
```

This updates `graph/generated.go` and `graph/model/models_gen.go`. Your resolver implementations in `schema.resolvers.go` are preserved.
